Notification pipelines

10.10.2016 notifications, messages, microservices

Introduction In this post I will share some thoughts upon implementing a notification pipeline in a microservice architecture. It should be said though that I don’t yet have much practical experiences, this post is rather designed as a discussion about the topic. While working on the OMS for AEGEE, I found that this topic is actually quite a tough point when developing a distributed application. A lot of microservices will want to generate notifications for different things, resulting in a high number of notifications and a high diversity Users want to control how and which notifications they want Notifications need to be managed centrally, as it would get inconsistend having the notification settings everywhere Certain events could potentially trigger a high notification throughput (a huge event gets canceled, popular news post) There are several channels how notifications can be distributed: Mail, Pop-up notification, Push-notifications, Chat integration, … The good thing is that this topic is not necessarily time-critical.
